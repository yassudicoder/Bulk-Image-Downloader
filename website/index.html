Debrief
Q&A · 6
Q1Is it free?
Yes — every feature is unlocked during the beta, with no paywall and no gating. A few power features are marked "Pro" for later, but they're open to everyone right now.
Q2Do you see my images?
No. Scanning, de-duplication and ZIP building happen entirely on your device. There is no server to send images to.
Q3Which pages does it work on?
Any normal web page. Browser-internal pages and the Chrome Web Store are off-limits to every extension — that's a browser rule, not our choice.
Q4Where do downloads go?
Into your browser's Downloads folder, like any other download — or tidy subfolders, if you set up folder rules.
Q5How does dedupe work?
Each image gets a tiny 64-bit perceptual fingerprint. Images with close fingerprints are grouped, and you choose which copy to keep — usually the largest.
Q6What permissions does it need?
Just the active tab, only when you click the icon. To fetch images from servers that block cross-origin requests, it can ask for one optional "all sites" grant — but only if some images need it, and only with your say-so.

No Data Leaves Device

Your images never leave your machine.

No account to create, no server to phone home to. The extension reads a page only when you click it, does its work locally, and forgets everything when the tab closes.

  • No uploads
  • No tracking
  • No account
  • On-device processing
  • Active-tab, on click only
  • Opt-in analytics, off by default
On analyticsIf you switch usage stats on, they're anonymous and content-free — counts and feature names only, never URLs, filenames or image data.
